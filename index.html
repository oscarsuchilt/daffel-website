Flujo de Renta Garantizada

Integración Belong × Nowo para garantizar renta segura sin riesgos para host ni roomie.

!

Regla de oro

El Seguro Nowo se paga adelantado, pero solo cuando la renta ya está confirmada. El roomie nunca pierde dinero si el host rechaza.

Los tres caminos

B. Host acepta la reserva manualmente
C. Host rechaza la reserva

Acción del Host

El host publica con nivel AAA

Completa el formulario de Belong y activa el Seguro Nowo para su propiedad.

  • Tipo de espacioDepartamento, casa o loft. Privado, entero o compartido.
  • Nivel AAASeguro Nowo + primer mes de renta + depósito en garantía.
  • Datos para NowoTipo de contrato, disponibilidad, cuenta bancaria y RFC del administrador.
  • Modo de aprobaciónAutomática o por aprobar manualmente. Define qué escenario se activará.

Resultado

Propiedad publicada con badge Nowo Garantizado • AAA.

Por aprobar manualmente

Acción del Roomie

El roomie encuentra la propiedad

Navega el catálogo de Belong, ve el badge AAA y aplica solo o en pool.

  • Explora el catálogoVista lista o mapa. Filtra por ciudad y precio. Ve el badge Nowo Garantizado.
  • Aplica o crea un poolSolo o con otros roomies. Belong verifica que el perfil tenga CURP, ocupación y domicilio.
  • Datos enviados a NowoCURP · Ocupación · Domicilio · m² · Monto de renta — automático vía API.

Bifurcación

Roomie aprobado — ¿qué escenario?

El flujo se divide según el modo que el host configuró en su propiedad.

→
Escenario A — Aprobación automática
El roomie puede rentar de inmediato. Sin intervención del host.
→
Escenario B — Host acepta
La propiedad requiere aprobación manual. El host acepta la reserva.
→
Escenario C — Host rechaza
La propiedad requiere aprobación manual. El host decide no aceptar.
Escenario A

Roomie aprobado — renta directa

Modo automático. El roomie renta sin que el host intervenga.

  • Notificación al roomieNowo te aprobó. Puedes rentar ahora. Se muestra el costo del seguro (~$6,000 MXN / 12 meses).
  • Roomie confirma la rentaToca "Rentar propiedad". En este momento la renta queda confirmada y la propiedad reservada.
  • Pago del Seguro NowoPago adelantado inmediato a través del motor de pagos de Nowo.
  • Póliza emitidaCobertura 100%: renta mensual garantizada + cobertura jurídica + garantía de servicios.

Escenario B

El host acepta la reserva

Nowo aprobó al roomie. El host revisa y acepta. El roomie no ha pagado nada todavía.

  • Notificación al hostVe el perfil del roomie: foto, compatibilidad, hobbies, ocupación y confirmación Nowo.
  • Host acepta la reservaToca "Aceptar reserva". Renta confirmada. Belong notifica al roomie.
  • Roomie paga el Seguro NowoSolo en este momento — tras la aceptación — se realiza el pago adelantado. Tiempo límite: 24-48 h.
  • Póliza emitida + propiedad rentadaHost recibe primer mes + depósito. Contrato digital firmado. Renta garantizada activa.

Si el roomie no paga en el tiempo límite

La reserva se cancela automáticamente. La propiedad vuelve a estar disponible. El host no tiene perjuicio alguno.

Escenario B — Renta confirmada

Póliza emitida — Propiedad rentada

El roomie pagó el seguro tras la aceptación del host. Renta garantizada activa.

Riesgo para el roomie

Ninguno antes de la aceptación. El pago solo ocurre cuando la renta ya es un hecho.

Riesgo para el host

Ninguno. El seguro Nowo garantiza el pago mensual aunque el inquilino incumpla.

  • Propiedad marcada OcupadaOtros roomies ya no pueden aplicar a esa habitación.
  • Contrato digital firmadoDentro de la plataforma Belong.
  • Renta activa mes a mesCon respaldo de Nowo y Aseguradora Insurgentes.
Escenario C

El host rechaza la reserva

Nowo aprobó al roomie, pero el host no acepta. El roomie no pierde nada.

0

Ningún cargo económico al roomie

El seguro se paga solo si hay aceptación confirmada. Sin perder nada.

  • Host registra motivo de rechazoSelecciona una razón. Queda registrado en Belong.
  • Aprobación Nowo sigue vigenteEl roomie puede aplicar a otra propiedad AAA sin nueva evaluación. Vigencia: 30-60 días (a definir).
  • Belong sugiere propiedades similaresEl roomie recibe recomendaciones de otras propiedades disponibles.
  • Posible penalización al hostSi rechaza recurrentemente sin justificación: advertencia, reducción de visibilidad o suspensión del badge Nowo.
Resumen

Los tres escenarios

Vista comparativa de los tres caminos posibles y sus resultados.

A — Automático

Sin intervención del host. El roomie confirma, paga el seguro y renta en minutos. Riesgo cero para ambos.

B — Host acepta

El host revisa y acepta. Solo tras la aceptación el roomie paga el seguro. Riesgo cero antes de aceptar.

C — Host rechaza

El roomie no paga nada. Su aprobación Nowo sigue vigente. El host puede ser penalizado si rechaza sin justificación.
